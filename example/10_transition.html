⚡ Transition 기본 개념

📖 Transition 이론

Transition은 상태 변화를 부드럽게 만들어주는 CSS 속성입니다.

❌ Transition 없으면:

색상/크기가 "뚝" 하고 바뀜 (딱딱한 느낌)

✅ Transition 있으면:

색상/크기가 부드럽게 변화 (자연스러운 느낌)

Transition 속성:

  • • transition → 전체 전환
  • • transition-colors → 색상만
  • • transition-transform → 변형만
  • • transition-opacity → 투명도만

Duration (속도):

  • • duration-150 → 150ms (빠름)
  • • duration-300 → 300ms (적당)
  • • duration-500 → 500ms (느림)

Easing (곡선):

  • • ease-linear → 일정
  • • ease-in → 시작 느림
  • • ease-out → 끝 느림
  • • ease-in-out → 자연스러움

Delay (지연):

  • • delay-75 → 75ms 후 시작
  • • delay-150 → 150ms 후 시작
  • • delay-300 → 300ms 후 시작

⏱️ Duration — 전환 속도

📖 Duration 이론

Duration은 전환이 얼마나 오래 걸릴지 지정합니다.

빠른 전환:

  • • duration-75 → 75ms
  • • duration-100 → 100ms
  • • duration-150 → 150ms

느린 전환:

  • • duration-500 → 500ms
  • • duration-700 → 700ms
  • • duration-1000 → 1000ms (1초)

추천:

  • • duration-200 → 버튼/링크
  • • duration-300 → 카드/박스

Tip:

  • • 너무 빠르면 안 보임
  • • 너무 느리면 답답함
  • • 150~300ms가 적당

📈 Easing — 전환 곡선

📖 Easing 이론

Easing은 전환이 어떤 속도 곡선으로 진행될지 결정합니다.

Easing 종류:

  • • ease-linear → 균일한 속도
  • • ease-in → 느리게 시작
  • • ease-out → 느리게 종료
  • • ease-in-out → 자연스러움

Tip:

  • • ease-out: 카드, 모달 등장
  • • ease-in: 요소 사라짐
  • • ease-in-out: 일반적 사용
  • • linear: 로딩 스피너

⏳ Delay — 전환 지연

📖 Delay 이론

Delay는 전환이 시작되기 전 대기 시간입니다.

Delay 값:

  • • delay-75 → 75ms 후
  • • delay-100 → 100ms 후
  • • delay-150 → 150ms 후
  • • delay-300 → 300ms 후

Tip:

  • • 순차적 애니메이션
  • • 툴팁/드롭다운 등장
  • • 리스트 항목 순서대로 나타남
