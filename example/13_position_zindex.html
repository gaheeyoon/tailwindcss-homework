📌 Position 기본 개념

📖 Position 이론

Position은 요소의 배치 방식을 결정합니다.

Position 종류:

  • • static → 기본값 (위치 지정 X)
  • • relative → 기준점 (자식의 부모)
  • • absolute → 부모 기준 고정
  • • fixed → 화면에 고정
  • • sticky → 스크롤 시 고정

위치 지정:

  • • top-0, top-4 → 위에서 거리
  • • right-0, right-4 → 오른쪽
  • • bottom-0 → 아래쪽
  • • left-0 → 왼쪽
  • • inset-0 → 상하좌우 모두 0

Tip:

  • • relative + absolute (배지, 오버레이)
  • • fixed (헤더, 버튼, 알림)
  • • sticky (네비게이션, 제목)

중요 팁:

  • • relative는 absolute의 기준점
  • • fixed는 스크롤과 무관
  • • z-index로 쌓임 순서 제어

📌 Fixed — 화면에 고정

📖 Fixed 이론

Fixed는 스크롤과 무관하게 화면에 고정됩니다.

주요 사용:

  • • fixed top-0 → 상단 헤더
  • • fixed bottom-4 right-4 → 플로팅 버튼
  • • fixed inset-0 → 모달 배경

Tip:

  • • 고정 헤더/푸터
  • • 플로팅 액션 버튼 (FAB)
  • • 알림 토스트
  • • 모달 오버레이

실습 7: Fixed 헤더 (시뮬레이션)

📝 실습 과제

  • • fixed + top-0 + left-0 + w-full
  • • 실제로는 스크롤해도 고정됨

📌 Sticky — 스크롤 시 고정

📖 Sticky 이론

Sticky는 relative와 fixed의 중간 성격을 가집니다. 스크롤 시 특정 위치에 도달하면 고정됩니다.

사용 방법:

  • • sticky top-0 → 상단 도달 시 고정
  • • sticky top-20 → 20px 떨어진 곳에 고정
  • • 부모 요소 벗어나면 다시 스크롤됨

Tip:

  • • 섹션 제목
  • • 테이블 헤더
  • • 사이드바 네비게이션
  • • 필터/검색 바

🎚️ Z-index — 쌓임 순서

📖 Z-index 이론

Z-index는 요소의 쌓임 순서를 제어합니다. 숫자가 클수록 앞에 표시됩니다.

Tailwind 기본값:

  • • z-0 → 0
  • • z-10 → 10
  • • z-20 → 20
  • • z-30 → 30
  • • z-40 → 40
  • • z-50 → 50
  • • z-[999] → 임의값

일반적으로:

  • • z-0 ~ z-10: 일반 콘텐츠
  • • z-20 ~ z-30: 배지, 툴팁
  • • z-40: 드롭다운, 팝오버
  • • z-50: 모달, 오버레이
  • • z-[999]: 최상위 (토스트)

실습 11: Z-index 기본

📝 실습 과제

  • • 세 개의 박스를 겹치게 배치
  • • z-10, z-20, z-30 적용
  • • 숫자 클수록 앞에 표시
